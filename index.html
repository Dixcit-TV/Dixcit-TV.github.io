About Me

I am a junior game programmer currently in my 3rd year of Game Development Bachelor at Digital Art and Entertainment in Kortrijk, Belgium. I am still figuring out which programming area I would like to focus on but I have a great interest for Graphics and Engine programming. I am also always keen on procedural generation.

Little bit of backstory...

Playing video games as a teenager grew my curiosity towards programming. All those virtual worlds, stories, mechanics, etc... were fascinating and I was imagining being able to do the same, create intricates virtual adventures people across the globe would enjoy and get inspired by.

I started my journey as a programmer by following and graduating a Bachelor in Software Development. After 3,5 years working as a software developer on automated property management solutions for servicers, I was assured that I picked the right carrier path but didn't feel that I was currently in the right industry as I didn't relate to it or felt invested in the product.
This led me to decide to specialize in the industry that grew my curiosity to programming: the gaming industry; and enrol for Digital Arts and Entertainment.

THE OUTER WALK

For this project, I wanted to experiment in creating a real-time planet generator on the GPU. The goal was to allow the user to create planets on the fly, using only sliders and color pickers and then visit their creation. Aside from being a school project, I took it as an opportunity to jump out of my comfort zone and try out something new. This project is developed using a DAE Framework based on DX11 and the PhysX API, as well as IMGUI.

View Project

N-GENIUS - 2D ENGINE

A first attempt at a simple 2D engine. It was a good introduction to different programming patterns and caveats in developping an engine. I experimented with component patterns, ways to implement a simple and clean observer patterns to limit coupling, and ways to efficiently handle resources, audio and inputs. I then implemented a few levels of the game QBert as proof of Concept. The engine is developed in C++ using the glm library and SDL.

SOFTWARE RASTERIZER MEETS DX11

After being introduced to rasterizing by developing a software rasterizer then a hardware rasterizer using DX11 (with Effects framework), my final Graphics programming 1 project was to merge them in one project allowing to switch between them at runtime. Each rasterizer having to handle basic features such as culling modes, transparency, diffuse and specular shading, and normal mapping.

View Project

RAYTRACING IN C++

My first fully functionning right-handed software raytracer written in C++. I implemented a dynamic camera with zoom and movement controls based on UE4. The raytracer handles spheres, planes and triangle meshes, point and directional lights, culling modes and several BRDFs such as Lambert diffuse, Phong and Cook-Torrance.

ZELIORA - A 2D DUNGEON CRAWLER

As final project for the (C++) Programming course, we are tasks to re-create a level of our favorite childhood game or invent our own. I decided to create a small dungeon crawler in the style of Final Fantasy. My motivation was to exercise new techniques, knowledge or algorithm. I worked on procedural 2D dungeon generation, how to implement a menu system, inventory, and path finding to have some AI crawling around. The game was developed in C++ using SDL and OpenGl.
